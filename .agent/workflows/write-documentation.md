# Trợ lý Viết Tài liệu

Hướng dẫn tôi tạo ra tài liệu toàn diện cho các thành phần nền tảng ZenSpa.

## Bước 1: Thu thập Ngữ cảnh

- Loại tài liệu (README, API docs, hướng dẫn setup, hướng dẫn người dùng, hướng dẫn triển khai)
- Đối tượng mục tiêu (developers, users, admins, stakeholders)
- Phạm vi (thành phần đơn lẻ, hệ thống đầy đủ, feature cụ thể)
- Tài liệu hiện có để tham chiếu hoặc cập nhật
- Nguồn thông tin chính (code, requirements, design docs)

## Bước 2: Phân tích Yêu cầu

- Xác định mục tiêu tài liệu và tiêu chí thành công
- Xác định phần must-have (tổng quan, cài đặt, sử dụng, ví dụ)
- Đánh giá độ phức tạp kỹ thuật và kiến thức tiên quyết
- Cân nhắc nhu cầu quốc tế hóa (tiếng Việt chính)
- Lập kế hoạch bảo trì và cập nhật

## Bước 3: Cấu trúc Tài liệu

- Tạo hệ thống phân cấp phần logic
- Định nghĩa mục lục
- Thiết lập định dạng và phong cách nhất quán
- Lập kế hoạch tham chiếu chéo và điều hướng
- Bao gồm siêu dữ liệu (phiên bản, cập nhật cuối, tác giả)

## Bước 4: Thu thập Nội dung

- Trích xuất thông tin chính từ tài liệu nguồn
- Thu thập ví dụ code và đoạn mã
- Tài liệu tùy chọn cấu hình và tham số
- Bao gồm phần khắc phục sự cố và FAQ
- Thu thập yếu tố trực quan (sơ đồ, screenshots nếu áp dụng)

## Bước 5: Viết Nội dung

- Bắt đầu với tổng quan rõ ràng, súc tích
- Cung cấp hướng dẫn từng bước
- Bao gồm ví dụ thực tế và trường hợp sử dụng
- Giải thích khái niệm với mức độ chi tiết phù hợp
- Sử dụng thuật ngữ và định dạng nhất quán

## Bước 6: Thêm Yếu tố Hỗ trợ

- Tạo sơ đồ mermaid cho kiến trúc/luồng quy trình
- Thêm làm nổi bật cú pháp code
- Bao gồm bảng cho cấu hình/so sánh
- Cung cấp liên kết đến tài liệu liên quan
- Thêm siêu dữ liệu và thông tin phiên bản

## Bước 7: Review và Xác thực

- Kiểm tra tính hoàn chỉnh và chính xác
- Xác thực tất cả liên kết và tham chiếu
- Đảm bảo định dạng và phong cách nhất quán
- Test hướng dẫn trên môi trường mới
- Nhận phản hồi từ thành viên nhóm

## Bước 8: Xuất bản và Bảo trì

- Lưu vào vị trí phù hợp trong cấu trúc docs/
- Commit vào version control với thông điệp rõ ràng
- Cập nhật bất kỳ liên kết tài liệu liên quan
- Lập kế hoạch review và cập nhật thường xuyên
- Thiết lập quyền sở hữu cho bảo trì tương lai

## Loại Tài liệu

### File README

- Tổng quan dự án và mục đích
- Hướng dẫn bắt đầu nhanh
- Hướng dẫn cài đặt
- Ví dụ sử dụng cơ bản
- Hướng dẫn đóng góp
- Giấy phép và thông tin liên hệ

### Hướng dẫn Setup

- Chuẩn bị môi trường
- Cài đặt phụ thuộc
- Bước cấu hình
- Thủ tục xác minh
- Khắc phục vấn đề phổ biến

### Tài liệu API

- Thông số kỹ thuật endpoint
- Ví dụ request/response
- Yêu cầu xác thực
- Xử lý lỗi
- Giới hạn tốc độ và hạn ngạch

### Hướng dẫn Người dùng

- Giải thích feature
- Hướng dẫn workflow
- Thực tiễn tốt nhất
- Cấu hình nâng cao
- Hướng dẫn tích hợp

### Hướng dẫn Triển khai

- Yêu cầu cơ sở hạ tầng
- Thủ tục triển khai
- Setup monitoring
- Sao lưu và khôi phục
- Cân nhắc mở rộng

## Tiêu chuẩn Chất lượng

### Chất lượng Nội dung

- Ngôn ngữ rõ ràng, súc tích (tiếng Việt chính)
- Luồng logic và tổ chức
- Thông tin hoàn chỉnh và chính xác
- Độ sâu kỹ thuật phù hợp
- Thuật ngữ nhất quán

### Chất lượng Kỹ thuật

- Ví dụ code hoạt động
- Mẫu cấu hình hợp lệ
- Đường dẫn file và lệnh chính xác
- Phiên bản phụ thuộc cập nhật
- Thủ tục đã test

### Chất lượng Trình bày

- Định dạng nhất quán
- Hệ thống phân cấp tiêu đề đúng
- Làm nổi bật cú pháp cho code
- Liên kết nội bộ/bên ngoài hoạt động
- Ngoại hình chuyên nghiệp

Hãy cho tôi biết loại tài liệu và mục tiêu khi bạn sẵn sàng bắt đầu viết tài liệu.
